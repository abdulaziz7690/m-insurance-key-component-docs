Component

Check Claim Process

The process to check user claim status from manulife, so user is able to see the claim details and amount that user must to pay.

Overview

The application collect claim number and taxonomy that inputted by user, and validating it to the next process that is calling check claim api:

  1. The validation process is to check acceptance criteria for claim number and taxonomy.
  2. Calling check claim api to match the claim number and taxonomy to the server.

How to?

In this section, we are going to go through necessary steps of check claim process.

The flow starts with user inputting claim number and selecting available taxonomy in the fields provided, when user taps on SUBMIT button the application will validating it.

Check Claim Page with claim number and taxonomy has been inputted


Below is the components of check claim page:


Check Claim Page Components

  1. Claim Number Field: User input claim number here, default input type is text.
  2. Taxonomy Drop Down: User select taxonomy here
  3. Submit Button: The submit button will validate the claim number and taxonomy which has been inputted by user before. If the validation is passed, next the check claim api function will be called.

Below is the validation scenarios for check claim process:

1.1 User Leaves Fields Empty

The user will be able to see inline error messages if user taps on SUBMIT button without entering any values on the fields.

Inline Error User Leaves Both of Fields Empty Scenario

Inline Error User Leaves Taxonomy Drop Down Empty Scenario

The inline error text are "Please fill the mandatory fields." and "Please select the taxonomy code." incorrect text fields or drop down will be highlighted with red color.


1.2 Incorrect Claim Number Format

The user will be able to see inline error messages if user taps on SUBMIT button with invalid claim number format.

Inline Error Incorrect Claim Number Format

The inline error text are "Please fill in the correct claim number." the incorrect text field will be highlighted with red color. The field should be inputted by number only.


1.3 The Combination of Claim Number and Taxonomy not found

The user will be able to see inline error messages if user taps on SUBMIT button with invalid combination of claim number and taxonomy.

Inline Error Incorrect Combination of Claim Number and Taxonomy

After validation process, the application will calling check claim api to find claim number and taxonomy to the server. If combination of claim number and taxonomy did not found, user will be able to see inline error "Sorry, the information you provided does not match our records. Please contact your Owner Services Team.".


1.4 The Combination of Claim Number and Taxonomy Was Found

Combination of Claim Number and Taxonomy Was Found

Claim Detail Page

After validation process, the application will calling check claim api to match claim number and taxonomy to the server. If claim number and taxonomy found, user will be able to see spinner masking the page, and redirected to claim detail page.

API responsible handling request by user who have entered their claim number and taxonomy. There are one API used in Check Claim Process.
