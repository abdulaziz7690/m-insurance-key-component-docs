Component

Login Process

The process by which an individual gains access to system by identifying and authenticating themselves by inputting email address and password.

Overview

The application collect email address and password that inputted by user, and validating it to the next process that is calling login api:

  1. The validation process is to check acceptance criteria for email address and password.
  2. Calling login api to match the email address and password to the server.

How to?

In this section, we are going to go through necessary steps of login process.

The flow starts with user inputting email address and password in the fields provided, when user taps on SIGN IN button the application will validating it.

Login Page with email address and password has been inputted


Below is the components of login page:


Login Page Components

  1. Email Field: User input email here, default input type is text.
  2. Password Field: User input password here, default input type is password.
  3. Password Watch: The password will reveal upon user clicked this password watch.
  4. Sign In Button: The sign in button will validate the email address and password which has been inputted by user before. If the validation is passed, next the login api function will be called.

Below is the validation scenarios for login process:

1.1 User Leaves Fields Empty

The user will be able to see inline error messages if user taps on SIGN IN button without entering any values on the fields.

Inline Error User Leaves Both of Fields Empty Scenario

Inline Error User Leave Email Fields Empty Scenario

Inline Error User Leave Password Fields Empty Scenario

The inline error text are "Please enter your email address and your password." the incorrect text fields will be highlighted with red color.


1.2 Incorrect Email Address Format

The user will be able to see inline error messages if user taps on SIGN IN button with invalid email format.

Inline Error Incorrect Email Address Format

The inline error text are "Please re-enter your email address. Remember, the valid format is “email@domain.com”" the incorrect text field will be highlighted with red color.


1.3 Incorrect Email or Password

The user will be able to see inline error messages if user taps on SIGN IN button with invalid email or password.

Inline Error Incorrect Email or Password

After validation process, the application will calling login api to match email address and password to the server. If email address and password did not match, user will be able to see inline error "Please check your email and password.".


1.4 Correct Email and Password

Login Success

After validation process, the application will calling login api to match email address and password to the server. If email address and password match, user will be able to see spinner masking the page, and redirected to check claim page.

To achieve all features available in Login Process needs API integration to interact between application with database. API responsible handling request by user who have entered their email address and password. There are one API used in Login Process.
